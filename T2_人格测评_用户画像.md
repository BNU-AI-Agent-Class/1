# T2 · 用户画像

> 项目：人格测评补充说明 Agent

## 1. 主画像 · 四要素

| 要素 | 内容 |
|---|---|
| **是谁** | 小陈，19 岁，大一学生 |
| **在哪** | 晚上 10 点，宿舍床上，靠床帘挡光，用手机刷朋友圈 |
| **心情** | 有点犹豫，觉得每个选项都不太符合自己，好奇但又有点烦 |
| **目标** | 想认真做完这个人格测试，得到一个靠谱、能解释的结果 |

## 2. 一句话故事

> 小陈，19 岁，大一。晚上 10 点躺在床上刷朋友圈，看到同学转发的人格/情境测试链接。点开做到一题"我喜欢主动认识陌生人"，五个选项从"非常不同意"到"非常同意"，他觉得自己"不是主动找人，但别人找我也愿意聊"，没有一个选项能表达。他心里想说"算了随便选一个"，但又担心结果不准。

## 3. 现有的「土办法」及其不足

- **土办法**：随便选一个最接近的；或者多做几遍挑一个"顺眼"的结果；或者做完直接关掉不信。
- **不足**：测出来的画像和真实状态有偏差；得不到解释；浪费答题时间；错误画像可能误导后续决策。

## 4. HMW（把痛点翻译成设计问题）

1. 我们如何能，让小陈在手机上做人格测试时，当固定选项无法描述自己时，用自然语言补充说明并获得更贴合的维度理解？
2. 我们如何能，让小陈得到维度分数后，用一句话明白自己这个分数意味着什么，而不是只看一个标签？
3. 我们如何能，让小陈想"发挥自己特点"时，得到一个具体、可操作的小建议，而不是被告知"人格有缺陷需要修补"？

## 5. 次要画像

- 小王，21 岁，大三，找工作前做职业性格测试，想知道结果对求职有什么实际参考。
