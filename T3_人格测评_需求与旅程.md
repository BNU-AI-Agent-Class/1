# T3 · 需求三层与旅程地图

> 项目：人格测评补充说明 Agent

## 1. 需求三层

| 层 | 真需求 | 证据 | 它决定了产品的 |
|---|---|---|---|
| **刚性**（功能） | 当固定选项不准时，能用自然语言补充说明，获得更准确的人格维度分数和解释 | "我不是很喜欢主动说话，但是如果别人来找我，我会很愿意交流" | 核心功能：开放补充输入 + LLM 按规则赋分 + 维度解释 |
| **情感**（体验） | 不想被简单标签化，希望"我的复杂性能被理解"，不想被评判"你就是内向/外向" | "算了随便选一个"背后是对选项的无奈，怕被误读 | 语气：温和、不贴标签、承认人格的复杂性 |
| **成长**（长期） | 不只想要一次测评结果，而是想学会在不同场景下理解和运用自己的特点 | "适合的职业是什么？怎么发挥人格特点？" | 退场设计：给出可操作建议后，鼓励用户自己观察验证，而不是反复依赖测评 |

## 2. 旅程地图

| 阶段 | 用户动作 | 痛点 | 情绪 | AI 介入点 / 退场 |
|---|---|---|---|---|
| 进入 | 在朋友圈/群里看到测试链接，点开 | 不确定这个测试靠不靠谱 | 好奇 + 犹豫 | 欢迎语简短，说明"选项不准可以补充说明" |
| 答题 | 逐题选择 1-5 | 某些题目选项无法描述自己 | 困惑、有点烦 | 每题提供"以上都不符合？请补充"入口 |
| **关键：补充** | 点击补充，输入自然语言 | 不知道该怎么描述，或担心说多了没用 | 犹豫 → 被理解 | AI 接住补充说明："我理解了，你其实是…"；按规则赋分 |
| 看报告 | 查看五大人格各维度分数和解读 | 只看数字不懂含义；怕标签化 | 期待 + 疑惑 | 用通俗语言解释每个维度，强调"倾向不是标签" |
| **Agent 对话** | 向内置 agent 继续提问，如"我适合什么职业""我该怎么发挥优势" | 不知道问题是否越界；担心被敷衍 | 好奇 + 试探 | Agent 基于画像回答，守住"不诊断、不给唯一答案"的边界；可追问兴趣/能力等补充信息 |
| 结束 / 退场 | 获得建议后退出 | 担心结果不准或想反复测 | 平静或仍有疑虑 | 提示"这是当前状态的参考，不是永久标签"；建议在生活中观察验证 |

## 3. 两个必标点

- 🔴 **危机介入点**：在答题补充、看报告或 Agent 对话中，出现"不想活了""活着没意思""想伤害自己/他人"等表述 → AI 停止当前流程 → 给出危机热线/学校心理中心/辅导员资源，并建议联系真人。
- 🚪 **退场点**：用户获得解释和行动建议后，AI 不主动追问、不诱导继续聊；当用户反复纠结同一问题或要求"告诉我唯一正确答案"时，提示"人格是倾向不是定论，建议结合真实经历观察和验证"。
