# AI 赋分标准严格化修改建议

> 目标：把当前宽松的四步赋分法升级为严格版六步法，提升评分区分度与数据质量控制能力。
> 参考格式：`职业场景替换为学生场景建议.md`
> 修改原则：
> 1. 不改变 OCEAN 五个维度的测评方向；
> 2. 不改变 A/B/C/D 四个选项的分值梯度（A=4、B=3、C=2、D=1）；
> 3. 不改变 25 道题的题干和选项内容；
> 4. E 选项"以上都不符合，我想自己说明"及引导语保持不变。

---

## 一、当前赋分标准的问题诊断

| # | 问题 | 具体表现 | 导致的后果 |
|---|---|---|---|
| 1 | **基础分用「最匹配取整」** | 与 A 有 51% 相似就给 4 分，与 A 有 95% 相似也给 4 分 | 模糊回答和清晰回答得分相同，无区分度 |
| 2 | **关键词微调幅度过小且不累积** | 出现 3 个强化词和出现 1 个，调整量都只有 ±0.5 | 无法反映行为强度的真实差异 |
| 3 | **关键词词表存在重叠冲突** | `看情况、一半一半、有时候` 同时出现在减分词和混合词列表中 | LLM 无法判断应触发 weaken_penalty 还是 ambiguity |
| 4 | **未处理否定前缀导致语义反转** | "不讨厌社交"中 `讨厌` 被误判为减分词-强，实际语义相反 | 评分离谱：用户表达喜欢却被扣分 |
| 5 | **无相似度门槛** | 最高相似度只有 35% 也硬套最近选项的分数 | 低质量回答被当作正常回答处理 |
| 6 | **异常值兜底过于宽松** | 答非所问、敷衍回答直接给 2.5 分（正常中值） | 乱答的用户也能拿到"看起来正常"的分数 |
| 7 | **weak_match 与 off_topic 混为一谈** | 认真回答但偏离选项 vs 敷衍乱写，都强制 2.5 分 | 有意义的尝试和无效输入同等对待 |
| 8 | **维度定义过于简略** | 每维度仅一句话，LLM 无法在异常值时做合理推断 | 偏离选项的高质量回答只能给中值 |
| 9 | **缺少置信度输出** | LLM 不对自己的评分可靠性做判断 | 低质量评分无法被跟踪和拦截 |
| 10 | **LLM 存在位置偏见风险** | A 总是排第一（4 分），模型倾向给前面选项更高相似度 | 系统性高估高分选项的匹配度 |
| 11 | **关键词不区分语境方向** | "绝对不去"中"绝对"被误判为加分词（实际强化的是拒绝=低分方向） | 低分方向的强烈表达被误加分数，评分完全失准 |
| 12 | **否定前缀词表覆盖不全** | 口语化否定"别/甭/莫"不在词表中，"千万别"的"千万"被单独识别为加分词 | 口语场景下否定反转失效 |

## 二、改进一：基础分从「最匹配取整」改为「加权平均」

### 当前做法
```
最像 A → base_score = 4（不管相似度是 51% 还是 95%）
最像 B → base_score = 3
最像 C → base_score = 2
最像 D → base_score = 1
```

### 建议改为
```
base_score = (sim_A×4 + sim_B×3 + sim_C×2 + sim_D×1) / (sim_A+sim_B+sim_C+sim_D)
```

### 零分母保护

当四项相似度之和 < 1%（极端情况如输入纯乱码），分母为 0 导致公式失效。增加兜底逻辑：若四项相似度之和 < 1%，直接判定为 `off_topic`，final_score 取 2.5，不计算加权平均。

### 相似度门槛

| 条件 | 标记 | 处理 |
|---|---|---|
| 最高相似度 < 35% | `weak_match` | 用户认真回答但内容与所有选项偏离。**不强制 2.5**，而是基于维度行为锚点（见改进五）进行逻辑推断，给出 1.0–4.0 的推断分，confidence 标记为 < 0.5 |
| 最高与次高之差 < 10pp | `boundary` | 用户处于边界地带。按以下竞争规则处理 |
| 最高 ≥ 80% 且其余 < 20% | `strong_match` | 用户高度匹配某一个选项，评分可信度高。**跳过关键词微调**（此时加权平均已给出极明确的分数，微调只会引入噪声），直接以 weighted_base_score 为 final_score，quality_flag=valid，confidence=0.9，不触发二次评分 |

### boundary 多强竞争规则

"最高与次高之差 < 10pp → 取两选项分数中位数"不够精确，细化为三档：

| 情况 | 示例 | 处理 |
|---|---|---|
| 最高与次高之差 < 10pp，且第三与次高之差 ≥ 10pp | sim_A=55%, sim_B=48%, sim_C=20% | 取最高两选项分数的中位数（A/B → 3.5） |
| 最高与次高之差 < 10pp，且第三与次高之差 < 10pp | sim_A=35%, sim_B=33%, sim_C=30% | **三选项竞争**：取三者分数的平均值（A/B/C → 3.0） |
| 四项相似度两两之差均 < 10pp | sim_A=28%, sim_B=25%, sim_C=24%, sim_D=23% | 用户没有明确倾向 → `ambiguity=true`，base_score 取 2.5 |

### 位置偏见对策

LLM 存在位置偏见（position bias）——倾向于给排在前面的选项更高相似度。在提示词中增加以下指令：

> 在计算相似度之前，请先忽略选项的排列顺序（A/B/C/D 仅为编号，不代表偏好方向），仅基于语义内容进行判断。不要因为 A 排在第一位就倾向给出更高的相似度。

## 三、改进二：关键词微调从「统一 ±0.5」改为「三级强度累积 + 否定反转 + 语境方向判定 + 优先级控制」

### 当前做法
所有加分词统一 +0.5，所有减分词统一 -0.5，不累积，不处理否定前缀，不区分语境方向。

### 建议改为：修正后的完整词表

#### 加分词（三级）

**强（weight=1.5）**：
甚至、超额、必须、立刻、绝对、极度享受、完全无法忍受、毫无疑问、完全、极其、总是、一定

**中（weight=1.0）**：
非常、特别、格外

**弱（weight=0.5）**：
很喜欢、很期待、很享受、很感兴趣、比较喜欢、倾向于

#### 减分词（三级）

**强（weight=1.5）**：
抗拒、讨厌、不喜欢、懒得、完全不、绝不、打死也不、被迫、不得已

**中（weight=1.0）**：
偶尔、尽量、看心情、不得不、有点、虽然、但是、有时、可能、大概、勉强、不太想、不情愿

**弱（weight=0.5）**：
不太确定

#### 混合词（出现即触发 ambiguity=true）
有时候、看情况、一半一半、视情况而定、时好时坏、不一定、摇摆不定、忽上忽下

### 词表重叠修正

旧版中以下词同时出现在「减分词-弱」和「混合词」两个列表中，造成 LLM 判断冲突。修正：将重叠词**只保留在混合词中**，并从减分词-弱中移除：

| 词 | 旧版位置 | 修正后位置 | 理由 |
|---|---|---|---|
| `看情况` | 减分词-弱 **且** 混合词 | 仅混合词 | 这些词的核心语义是"不确定/摇摆"，不是"行为强度降低"。触发 ambiguity 比累计 weaken_penalty 更准确 |
| `一半一半` | 减分词-弱 **且** 混合词 | 仅混合词 | 同上 |
| `有时候` | 减分词-弱 **且** 混合词 | 仅混合词 | 同上 |
| `不一定` | 减分词-弱 **且** 混合词 | 仅混合词 | 同上 |
| `视情况而定` | 减分词-弱 **且** 混合词 | 仅混合词 | 同上 |
| `时好时坏` | 减分词-弱 **且** 混合词 | 仅混合词 | 同上 |

### 优先级规则

**混合词优先于加减分词。** 任何混合词匹配 → ambiguity=true，该词不再计入 intensity_boost 或 weaken_penalty。当 ambiguity=true 且最高与次高相似度之差 < 10pp 时，取相邻两选项分数的中位数作为基础分。

### 否定前缀反转规则

纯关键词匹配无法处理"我不讨厌社交"这类表达。增加以下规则：

| 否定模式 | 示例 | 处理 |
|---|---|---|
| 不/没/无 + 关键词 | "不讨厌""没抗拒""无所谓" | 关键词不计入原方向，反转计入反方向（"不讨厌"→加分词-弱, weight=0.5） |
| 不太/不是很 + 关键词 | "不太喜欢""不是很抗拒" | 反转但减弱一级（"不太喜欢"→不匹配减分词-强/中，仅计入减分词-弱） |
| 双重否定 | "不是不喜欢""没办法不去" | 还原为正向，按原方向正常计 |

此项规则须在 LLM 提示词中**显式写明**，而非仅依赖关键词匹配列表。系统提示中应加入：

> 在进行关键词匹配之前，先检查用户文本中是否存在否定结构（不/没/无/别/甭/莫/不太/不是很/不怎么/并不/从未/绝不/决不/不要/不能/不准）。如果关键词前有否定前缀，必须反转该关键词的加减方向并降一级权重。例如"不讨厌"不应识别为减分词，而应识别为加分词-弱。检查时从最长复合否定开始匹配（避免"绝不"被"不"单字截断）。

### 语境方向判定规则（Keyword Direction by Context）

**核心问题**：同一个关键词在不同语境下可能强化相反的方向。"绝对不去"中"绝对"强化的是**拒绝新体验**（低分方向），而"绝对要去"中"绝对"强化的是**拥抱新体验**（高分方向）。纯关键词极性匹配会将两者都计入加分词，导致方向性误判。

**判定方法**：

1. 先确定用户回答的**核心行为方向**（由加权平均 base_score 指示）：base_score > 2.5 为高分倾向，base_score < 2.5 为低分倾向
2. 判断关键词是**强化了该方向**还是**对抗了该方向**：
   - 关键词强化了核心行为方向 → 维持原极性（加分词仍加分，减分词仍减分）
   - 关键词对抗了核心行为方向 → **反转极性**（加分词反转为减分词，减分词反转为加分词，weight 不变）

**效果对照**：

| 示例 | base_score | 关键词 | 行为方向 | 判定 | 处理 |
|---|---|---|---|---|---|
| "绝对要去" | 3.8（高分） | "绝对" | 强化高分方向 | 维持：加分词-强 | +1.5 ✅ |
| "绝对不去" | 1.3（低分） | "绝对" | 强化低分方向 | 反转：变为减分词-强 | -1.5 ✅ |
| "完全按计划" | 3.7（高分） | "完全" | 强化高分方向 | 维持：加分词-中 | +1.0 ✅ |
| "完全看心情" | 1.2（低分） | "完全" | 强化低分方向 | 反转：变为减分词-中 | -1.0 ✅ |
| "非常不喜欢" | 1.1（低分） | "非常"+"不喜欢" | 均强化低分方向 | 先否定反转"不喜欢"→加分词-弱(+0.5)，再语境判定"非常"强化低分→反转为减分词-中(-1.0)，net=-0.5 | ✅ |

系统提示中应加入：

> 关键词的加减分方向不是绝对的。在判断关键词方向之前，先用加权平均的 base_score 确定用户回答的核心行为方向（>2.5 为高分倾向，<2.5 为低分倾向）。如果关键词修饰的行为本身是低分方向的行为，则关键词的加减方向应当反转。例如：base_score=1.3（低分倾向），"绝对"强化的是低分方向 → 从加分词反转为减分词，按原 weight 计入 weaken_penalty。

### 微调计算公式（累积，幅度上限 ±1.5）

```
intensity_boost = Σ(加分词出现次数 × 该级 weight)    // 先执行否定反转再统计
weaken_penalty  = Σ(减分词出现次数 × 该级 weight)    // 先执行否定反转再统计
net_adjustment  = min(1.5, intensity_boost) - min(1.5, weaken_penalty)
final_score     = base_score + net_adjustment
```

### 效果对照

| 示例 | 旧版 | 新版（含否定反转+词表修正） | 效果 |
|---|---|---|---|
| "我一定立刻去，非常期待" | +0.5 | 强×2(1.5×2) + 中×1(1.0) = 上限 +1.5 | 强烈表达充分区分 |
| "有点想但又不太确定" | -0.5 | 中减×1 → -0.5；"不太确定"为混合词，触发 ambiguity | 摇摆被准确识别 |
| "完全不喜欢，非常抗拒" | -0.5 | 强加×1 + 强减×2 → ambiguity=true，取中位数 | 自相矛盾被识别 |
| "我不讨厌社交" | `讨厌`→减分词-强，扣 1.5 | 否定反转："不讨厌"→加分词-弱，+0.5 | **修正：语义反转不再误判** |
| "看情况吧，有时候会" | `看情况`→减分词-弱，"有时候"→混合词，冲突 | 两个词均仅匹配混合词，ambiguity=true | **修正：不再产生冲突** |
| "绝对不去，太累了" | "绝对"→加分词-强，+1.5（致命误判） | base_score=1.3（低分方向）+语境方向判定→"绝对"强化低分方向，反转为减分词-强，-1.5→final≈1.0 | **新增：语境化方向不再误判** |
| "千万别让我动脑子" | "千万"→加分词-强，+1.5；"别"不在否定词表中 | 扩展否定词表覆盖"别"→"千万别"整体识别为否定结构，"千万"不单独计入 | **新增：口语否定不再漏网** |

## 四、改进三：质量标记（quality_flag）五类分级 + 信息密度考量 + 差异化处理

### quality_flag 完整定义

| quality_flag | 触发条件 | 处理方式 |
|---|---|---|
| `valid` | 回答与情境相关，语义清晰，最高相似度 ≥ 35% | 正常评分，分数完全采纳 |
| `weak_match` | 最高相似度 < 35%，但用户**确实在认真回答**（有实质内容，与情境相关） | **不强制 2.5**。基于维度行为锚点（见改进五）进行逻辑推断，给出 1.0–4.0 的推断分，confidence 强制标记为 < 0.5 |
| `boundary` | 最高与次高相似度之差 < 10pp | 按三强竞争规则（见改进一）取中位数或平均值 |
| `off_topic` | 答非所问、明显敷衍（如"随便""不知道""嗯嗯""。。。""哈哈"）、字数 < 5 个有意义汉字 | final_score 强制取 2.5。标记无效，维度总分计算时降权为 0.5（正常题目权重 1/5），并在报告中标注该维度因数据质量问题结果仅供参考 |
| `contradictory` | 文本内部自相矛盾（如"我很喜欢社交但也很讨厌人多的地方"），或关键词密度 > 40%（大量堆砌强化/弱化词，可能不真实） | ambiguity=true，取中间值 2.5。confidence 下调 0.2 |

### weak_match 与 off_topic 的关键区别

| 维度 | weak_match | off_topic |
|---|---|---|
| 用户意图 | 认真尝试回答 | 敷衍或乱写 |
| 回答质量 | 有实质内容，方向对但具体偏离 | 无实质内容 |
| 分数 | 基于锚点推断 1.0–4.0 | 强制 2.5 |
| 维度权重 | 正常 1.0 | 降权 0.5 |
| 用户反馈 | 告知得分及推断依据 | 提示重新描述 |
| confidence | < 0.5 | N/A（不适用） |

### 回答信息密度考量

在质量标记判定时增加启发式规则：

| 条件 | 调整 |
|---|---|
| 有效汉字数 ≥ 15 且关键词密度（关键词字数 / 总字数）< 10% | 回答有实质内容且未被关键词主导 → confidence 上调 0.1 |
| 有效汉字数 ≥ 10 但关键词密度 > 40% | 大量堆砌强化/弱化词，可能不真实或夸张 → 标记为 `contradictory` |

### 用户反馈差异化

| quality_flag | 向用户展示的反馈 |
|---|---|
| `valid` | 仅告知得分，不额外提示 |
| `weak_match` | 「你的回答与我预期的选项有一些差异。我根据你的描述做了推断，这道题计为 X 分。如果这个分数不太符合你的情况，可以重新描述一下吗？」 |
| `boundary` | 「你的回答似乎兼有两种倾向，这道题暂按 X 分计。我们继续下一题。」 |
| `off_topic` | 「你的回答与我预期的情境有些偏离，这道题暂按中间值计分。如果愿意，可以更具体地描述一下你的感受。」 |
| `contradictory` | 「你的回答里有一些看起来不太一致的地方，这道题暂按中间值计分。如果你愿意，可以再确认一下你的真实感受。」 |

### 报告中的体现

25 题完成后，报告中应包含：
1. 列出所有 quality_flag 非 valid 的题目及对应的 quality_flag 类型；
2. 标注 off_topic 题目对应的维度，提示该维度数据可信度受限；
3. 如果某个维度有 ≥2 题 quality_flag 非 valid，应在该维度的解释中增加"本维度数据质量较低，结果仅供参考"的说明。

## 五、改进四：评分置信度（confidence）分级 + 二次评分触发

### 基础置信度

LLM 必须对每条回答输出评分置信度（0.0–1.0）：

| 置信度 | 含义 | 处理 |
|---|---|---|
| ≥ 0.7 | 可信 | 正常采纳 |
| 0.5–0.7 | 存疑 | 分数采纳但标记，报告中标出供用户参考 |
| < 0.5 | 不可信 | 触发二次评分（见下方） |

### 信息密度对置信度的调整

在基础置信度之上应用调整：

| 条件 | confidence 调整 |
|---|---|
| 有效汉字数 ≥ 15 且关键词密度 < 10% | +0.1 |
| 关键词密度 > 40% | -0.2 |
| strong_match = true | +0.1 |
| boundary = true | -0.1 |

调整后 confidence 仍限制在 [0.0, 1.0] 范围内。

### 二次评分机制（轻量 inter-rater reliability）

```
第一次 LLM 评分（temperature=0.2）
    ↓
confidence < 0.5 或 quality_flag ≠ valid
    ↓
触发二次评分（同一 prompt，temperature 调至 0.5，增加随机性以暴露不确定性）
    ↓
比较两次评分：
  - 两次 final_score 之差 ≤ 0.5 → 取平均值，采纳。quality_flag 取两次中较优者
  - 两次 final_score 之差 > 0.5 → 取中值 2.5，quality_flag 标记为 "unreliable"
```

对于 confidence ≥ 0.7 且 quality_flag = valid 的正常回答，不触发二次评分。

## 六、改进五：维度行为锚点（替代简略的高低分定义）

### 当前做法
每维度仅一句话定义（如 O："好奇、喜欢新异与抽象 / 偏好熟悉、具体、可预期"），LLM 在用户回答完全跳出选项范围时，缺少足够信息做合理推断。

### 建议改为：四档行为锚点表

在第一步「情境锚定」中，向 LLM 提供以下完整锚点表，使其在 weak_match 和异常值场景下能够进行有理有据的推断：

| 维度 | 4 分锚点（极高） | 3 分锚点（偏高） | 2 分锚点（偏低） | 1 分锚点（极低） |
|---|---|---|---|---|
| **O 开放性** | 主动探索未知，享受新奇体验，喜欢抽象思辨，对陌生事物天然好奇 | 愿意尝试新事物，但不排斥熟悉的方案，在新鲜与安全之间保持开放 | 更偏好熟悉和可预期的事物，对变化持观望态度，需要有足够理由才会尝试 | 回避新鲜和不确定，坚持已知和常规，对变化有明显抵触 |
| **C 尽责性** | 自发制定计划并严格执行，追求卓越和完美交付，信守承诺，自我驱动 | 基本有条理，能按时完成任务，偶尔灵活调整但不影响整体质量 | 倾向于随性而为，计划执行力一般，对细节关注不多，需要外部提醒 | 明显拖延，不拘小节，对承诺较随意，最后期限才能推动行动 |
| **E 外向性** | 从社交中获得能量，喜欢成为焦点，主动发起对话和活动，乐于表达 | 能享受社交也能独处，在社交场合中适度表达，不排斥成为焦点但不主动寻求 | 偏好安静和小圈子，在社交中较被动，需要独处恢复能量 | 回避社交场合，需要大量独处时间，在人群中感到消耗而非充电 |
| **A 宜人性** | 优先考虑他人感受，高度共情，为维护关系和谐愿意退让，本能信任他人 | 在坚持自我和照顾他人之间保持平衡，友善但有底线 | 更重逻辑和原则，有时显得直接，对陌生人的善意保持警惕 | 优先自身利益，不惧冲突，较难信任他人，认为"好心往往被利用" |
| **N 神经质** | 情绪体验强烈且持久，容易焦虑、担忧和反复思考，小事也能引起较大波动 | 面对压力有明显波动，但随时间能逐渐平复，偶尔会多想但能拉回来 | 多数时候情绪稳定，压力下有反应但不持久，能较快恢复平静 | 情绪稳定，抗压能力突出，几乎不因小事波动，快速恢复平静 |

### 使用时机

- 当 quality_flag = `weak_match` 时：LLM 不再强制给 2.5，而是将用户回答与四档锚点做语义比对，选择最匹配档位的分数（4/3/2/1），confidence 标记为 < 0.5
- 当 quality_flag = `valid` 但 similarity 最高两项接近时：LLM 可参考锚点做微调确认，避免纯粹依赖关键词匹配
- 当用户回答中出现大量维度相关但非选项关键词时：锚点提供额外判断依据

## 七、改进六：跨题一致性检测（报告层面）

### 背景
同一维度有 5 道题。如果用户在第 11 题（E 外向性/周末恢复）写了"我特别喜欢和朋友出去玩"，但在第 12 题（E 外向性/陌生聚会）写了"我极度讨厌和陌生人说话"——两道题在同一维度方向相反，可能说明：
- 用户在**不同场景下表现确实不同**（合理的情况差异）
- 用户**对某些题目理解有偏差**（需要标注）

### 建议规则

25 题全部评分完成后，对同一维度内所有使用了 E 选项的题目做一次快速扫描：

| 条件 | 处理 |
|---|---|
| 该维度 ≥2 题使用 E 选项，且它们的 final_score 极差 ≥ 2.0 分（如一道 3.5，一道 1.0） | 报告中标注该维度"存在较大场景差异"，建议用户关注：同一人格特质在不同情境下可能有不同表现 |
| 该维度 ≥2 题使用 E 选项，极差 < 2.0 分 | 不额外标注，正常呈现 |
| 该维度仅有 0–1 题使用 E 选项 | 不触发扫描 |

### 重要说明

这不是打分修正——**不改变任何题目的分数**。仅作为报告层面的提示，帮助用户理解分数构成，体现"人格倾向不是标签"的理念。

## 八、修改步骤总结

| 步骤 | 旧版（四步法） | 新版（六步法） |
|---|---|---|
| 第一步 | 情境锚定（题目+选项） | 情境锚定（题目+选项+**维度行为锚点表**） |
| 第二步 | 语义相似度→最匹配取整 | 语义相似度→**加权平均**（含零分母保护 + 位置偏见指令 + **strong_match 跳过微调优化**） |
| 第三步 | 关键词 ±0.5（统一，不累积，不处理否定） | 关键词 **三级强度 × 权重累积**（±1.5 上限）+ **否定前缀反转**（扩展词表）+ **语境方向判定** + **混合词优先** + **词表去重** |
| 第四步 | 异常值处理 → 2.5 默认中值 | **质量标记**（5 类 quality_flag，含信息密度调整 + weak_match/off_topic 差异化） |
| 第五步 | — | **置信度输出**（含信息密度调整 + 二次评分触发机制） |
| 第六步 | — | **跨题一致性检测**（报告层面，不修改分数） |

### 检验发现：需要追加的 3 项修改

#### 追加 A：关键词需要「语境方向判定」（对应 Q1 低分、Q8 低分）

**问题复现**：

Q1 低分：「我**绝对**不去」→ "绝对"是强加分词（weight=1.5），但此处的"绝对"强化的是**拒绝新体验**（低 O 方向），而非高 O 方向。纯关键词匹配导致 +1.5 的致命误判。

Q8 低分：「**完全**看心情」→ "完全"是中加分词（weight=1.0），但此处强化的是**随性放任**（低 C 方向），而非高 C 方向。

**根因**：当前所有关键词的方向是固定的——加分词总是倾向高分方向，减分词总是倾向低分方向。但同一个词在不同语境下可能强化相反的方向。

**建议修改**：在关键词微调规则中增加「语境方向判定」条款：

> 关键词的加减分方向不是绝对的，必须结合该维度的方向进行判定。如果关键词修饰的行为**本身是低分方向的行为**，则关键词的加减方向应当反转：
>
> - "绝对不去"：关键词"绝对"修饰的是"不去"（该维度的低分行为），因此"绝对"应当从加分词反转为减分词（强化了低分方向的确认度），按原 weight 计入 weaken_penalty
> - "完全看心情"：关键词"完全"修饰的是"看心情"（该维度的低分行为），应反转为减分词
> - 反之："绝对要去"、"完全按计划执行"中，关键词修饰的是高分行为，维持原方向
>
> 判定方法：先确定用户回答的核心行为方向（由加权平均的 base_score 指示——base_score > 2.5 为高分倾向，< 2.5 为低分倾向），再判断关键词是强化了该方向（维持原极性）还是对抗了该方向（反转极性）。

**效果验证**：

| 样本 | 旧版关键词（误判） | 新版语境化判定（修正后） |
|---|---|---|
| Q1 低分 "绝对不去" | +1.5 加分词 | base_score≈1.3（低分方向），"绝对"强化低分方向 → 反转为减分词 → weaken_penalty+1.5 → final_score≈1.0 ✅ |
| Q8 低分 "完全看心情" | +1.0 加分词 | base_score≈1.2（低分方向），"完全"强化低分方向 → 反转为减分词 → weaken_penalty+1.0 → final_score≈1.0 ✅ |

---

#### 追加 B：否定前缀词表需要同时包含「别」「不要」「不」及其变体（对应 Q4 低分、Q15 低分）

**问题复现**：

Q4 低分：「**千万别**让我动脑子」→ 当前规则只覆盖"不/没/无/不太/不是很"，但"别"作为口语化否定未在词表中。"千万"被识别为加分词-强，"别"不触发否定反转，导致"千万别"整体被误加 1.5。

Q15 低分：「**绝不**发语音」→ 当前"绝不"在减分词-强中（weight=1.5），但"绝不当众说"（Q13 低分）中"绝不"的语义是强化回避行为（低 E）。虽然方向可能碰巧对，但缺乏系统性处理。

**建议修改**：扩展否定前缀列表：

> 否定前缀词表（检查顺序：先否定前缀，再关键词）：
> - 标准否定：不、没、无、非
> - 口语否定：别、甭、莫
> - 复合否定：不太、不是很、不怎么、并不、从未、绝不、决不
> - 禁止类：不要、不能、别去、不准

匹配时从最长复合否定开始检查（避免"绝不"被"不"单字截断）。

---

#### 追加 C：E 题中正常选择 A/B/C/D 的样本未被测试覆盖

**问题**：测试回答仅覆盖了选 E 开放说明的场景，但实际测评中大部分用户选 A/B/C/D。新标准主要优化的是 E 选项赋分，并未涉及 A/B/C/D 的直接计分逻辑（该部分确实不需修改），但需要确认一个边界场景：

场景：用户选 E 输入了一段文字，其语义与某个预设选项（如 B）高度匹配。新标准通过加权平均能准确给分，但旧标准的"最匹配取整"也大概率得到相同结果——**此时新标准的额外计算（加权平均+三级微调+quality_flag+confidence）是否引入了不必要的复杂度？**

**建议**：当 `strong_match = true`（最高相似度 ≥ 80% 且其余 < 20%）时，跳过三级关键词微调——因为此时加权平均已经给出了非常明确的分数，关键词微调只会引入噪声而不会改变实质结论。这样可以减少 ~40% 的计算复杂度（N 维度 strong_match 频率很高）。

简化后流程：
```
strong_match = true → quality_flag = valid, confidence = 0.9
                     跳过关键词微调，直接使用 weighted_base_score
                     不做二次评分
```

### 总体评价
