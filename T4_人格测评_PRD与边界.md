# T4 · PRD 与边界

> 项目：人格测评补充说明 Agent

## 1. 一页 PRD · 六要素

| 要素 | 内容 |
|---|---|
| ① 一句话定位 | 一个面向大学生的情境人格测评工具：25 道 A/B/C/D 情境题 + 选不出的题可补充自然语言说明，AI 按既定标准赋分，最终以雷达图呈现五大人格维度得分，并内置 Agent 供后续参考性问答 |
| ② 目标用户 | 小陈，19 岁，大一学生；晚上用手机刷朋友圈/群聊看到测试链接，做 25 道情境题时部分选项无法描述自己 |
| ③ 核心功能（≤3） | **1. 智能补充赋分**：25 道情境题每道 A/B/C/D，对选不出的题提供补充输入框，LLM 按既定赋分标准重新赋分并纳入维度计算；<br>**2. 雷达图维度报告**：基于答题+补充说明计算五大人格维度得分，以雷达图呈现，并给出通俗解释，避免标签化；<br>**3. 画像答疑 Agent**：用户可基于雷达图结果继续提问（如职业方向、学习风格、人际策略），Agent 结合画像给出参考性建议，不替代专业咨询 |
| ④ 明确不做 | 不诊断任何心理疾病；不开药/不治疗；不替代真人心理咨询；不处理危机（必须转介）；不给用户贴绝对化标签；不采集真实身份信息 |
| ⑤ 成功标准 | - 用户在遇到选项不匹配时，≥70% 选择使用补充说明而非随便选；<br>- 用户对雷达图维度解释的理解度自评 ≥4/5；<br>- Agent 对越界请求（诊断/治疗/自伤）100% 正确识别并拒绝/转介 |
| ⑥ 主要风险 | LLM 赋分偏离既定标准；用户对雷达图结果过度解读；标签化引发焦虑；漏识别危机信号；用户过度依赖 Agent 建议 |

## 2. 边界三问

- **能做什么？**
  - 展示 25 道情境选择题（A/B/C/D）；
  - 接收用户对选不出题目的自然语言补充说明，按既定 AI 赋分标准进行维度赋分；
  - 用雷达图展示五大人格维度得分，并用通俗语言解释；
  - 基于用户画像回答职业发展、学习风格、人际策略等一般性参考问题。

- **明确不做什么？**
  - 不诊断心理疾病、不判断"人格优劣"、不建议药物/治疗；
  - 不替代心理咨询师、职业规划师、医生等专业人士；
  - 不给用户贴"你必须是XX型"等绝对标签；
  - 不采集真实姓名、学号、联系方式等可识别身份信息。

- **何时必须转人？**
  - **触发信号**：用户表达自伤/自杀/伤害他人意图，或请求诊断/开药/治疗。
  - **动作**：立即停止当前测评/对话；声明"我无法处理这个问题"；给出危机热线/学校心理中心/就医建议。
  - **转给谁**：全国 24 小时心理援助热线、学校心理咨询中心、辅导员/家人、就近医院精神科。

## 3. 数据红黄绿清单

| 数据 | 灯 | 处理方式 |
|---|---|---|
| 真实姓名/学号/手机号 | 🔴 红 | 不采集 |
| 可识别身份的社交账号信息 | 🔴 红 | 不采集 |
| 真实咨询记录/医疗记录 | 🔴 红 | 不采集 |
| 用户答题选择（A/B/C/D）+ 补充文本 | 🟡 黄 | 化名/会话 ID 存储；仅用于本次测评与画像生成；明确告知用户；提供删除入口 |

| 25 道情境题及 A/B/C/D 选项 | 🟢 绿 | 项目自设/正常使用 |
| AI 赋分标准与维度计算规则 | 🟢 绿 | 项目自设/正常使用 |
| 公开心理学知识（大五人格解释等） | 🟢 绿 | 正常使用 |
| agent.md 长期画像配置文件 | 🟡 黄 | 本地/会话级存储；用户可查看、修改、删除；不上传云端 |

## 5. 人本 AI 七原则回扣

| 原则 | PRD 中如何体现 |
|---|---|
| 人在环 HITL | 危机/诊断/治疗请求必须由真人处理，AI 只转介 |
| 透明可解释 | 开场明确告知用户正在与 AI 交互；解释分数时引用用户原话 |
| 心智模型对齐 | 开场说明"不诊断、不治疗、结果仅供参考" |
| 优雅失败 | LLM 无法按标准赋分时承认局限，提示用户重新描述或选择最接近选项 |
| 信任校准 | 强调结果是"参考"而非"定论"，雷达图仅供自我理解 |
| 用户控制 | 随时可修改答案、可退出、可删除数据 |
| 伦理包容 | 不贴标签、不因人格类型歧视、保护隐私 |
